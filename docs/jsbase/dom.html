# 常见节点类型

  • 9 - DOCUMENT_NODE (window.document)
  • 1 - ELEMENT_NODE (body元素、a元素等)
  • 2 - ATTRIBUTE_NODE (class="xxx")
  • 3 - TEXT_NODE (文本节点)
  • 11 - DOCUMENT_FRAGMENT_NODE (document.createDocumentFragment())
  • 10 - DOCUMENT_TYPE_NODE (!DOCTYPE html)

# textcontent 和 innerText 区别

详情见此链接 (opens new window)

  1. innerText知道css。所以如果有隐藏文本，innerText会无视它，textContent不会
  2. 因为innerText关心css，它将触发一次重排，而textContent不会，所以innerText性能不好
  3. innerText无视script、style标签的Text节点

# 几何量与滚动几何量

# offsetParent

获取定位的父级，只读

  • 按照DOM树向上查找离调用该属性的元素最近的position属性不为static的祖先元素
  • 如果没找到，那么body元素就是offsetParent值
  • 如果在查找过程中，遇到td、th、table元素，那么就为这些值

# offsetTop

获取元素边框外沿到其定位父级边框内沿的上距离，读写

# offsetLeft

获取元素边框外沿到其定位父级边框内沿的左距离，读写

# offsetHeight

获取元素高（边框+填充+内容），只读

# offsetWidth

获取元素宽（边框+填充+内容），只读

# clientHeight

获取元素高（填充+内容），只读

# clientWidth

获取元素宽（填充+内容），只读

# scrollHeight

获取滚动元素的高度， 只读

如果滚动区域内的子节点比滚动区域小，那么该属性返回滚动区域的高度，使用clientHeight

# scrollTop、scrollLeft

element.scrollTop = 750;
element.scrollLeft = 750;
// 意味着到左边和上边750像素的区域在视区中不可见

# DOM中的JavaScript

# JavaScript默认同步解析

# defer

推迟外部脚本的下载与执行，直到HTML文档解析完成

# async

异步下载并执行JavaScript文件

  • 不会阻塞DOM的解析
  • 如果有多个async，那么执行顺序无序，先下载完的执行
  • async优先级比defer高
  • 如果使用document.createElement('script')创建的元素，并添加到DOM中，那么该脚本会按照async执行
  • script元素的onload、onerror、load、error事件，监听异步下载JavaScript文件的情况
