# 执行上下文

编译是把代码拿过来创建执行上下文，并创建变量环境、词法环境、可执行代码，将执行上下文压入执行栈中。

执行是在当前执行上下文环境下执行可执行代码。

var foo = 1;
function bar() {
    if(!foo) {
        // 相当于先var foo变量提升，foo为undefined
        var foo = 10;
    }
    console.log(foo);
}
bar(); // 10

# 对象池

使用对象池，防止过多的垃圾回收，避免内存泄漏

原来的做法：

function addVector(a, b) {
    let resultant = new Vector();
    resultant.x = a.x + b.x;
    resultant.y = a.y + b.y;
    return resultant;
}

如果这个函数频繁调用，那么程序也会频繁的进行垃圾回收

现在的做法：

function addVector(a, b, resultant) {
    resultant.x = a.x + b.x;
    resultant.y = a.y + b.y;
    return resultant;
}

let v1 = vectorPool.allocate();

# 尾调用优化

# 闭包

闭包指的是那些引用了另一个函数作用域中变量的函数，通常在嵌套函数中实现。

如果内部函数没有使用箭头函数定义，则this对象会在运行时绑定到执行函数的上下文。但是匿名函数在这种情况下不会绑定到某个对象上，这就意味着this会指向window

因为匿名函数永远不可能直接访问外部函数中的this和arguments

window.identity = 'the window';

let object = {
    identity: "my object",
    getIdentityFunc() {
        return function() {
            return this.identity;
        };
    }
};

console.log(object.getIdentityFunc()()); // the window

# Promise

# Promise.resolve()

如果传入的参数本身是一个期约，那他的行为类似于一个空包装，而且会保留传入期约的状态

let p = new Promise(() => {});
setTimeout(console.log, 0, p); // pending
setTimeout(console.log, 0, Promise.resolve(p)); // pending

# Promise.reject()

如果传入的参数本身是一个期约，不会像Promise.resolve()那样保留状态，这个期约会成为他返回拒绝期约的理由

# 查询字符串

let getQueryStringArgs = function() {
 // 取得没有开头问号的查询字符串
 let qs = (location.search.length > 0 ? location.search.substring(1) : ""),
 // 保存数据的对象
 args = {};
 // 把每个参数添加到 args 对象
 for (let item of qs.split("&").map(kv => kv.split("="))) {
 let name = decodeURIComponent(item[0]),
 value = decodeURIComponent(item[1]);
 if (name.length) {
 args[name] = value;
 }
 } 
    return args;
}
